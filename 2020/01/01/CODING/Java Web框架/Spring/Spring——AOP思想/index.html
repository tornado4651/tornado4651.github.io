概述

AOP的概念

AOP的特点

  1. 降低模块之间的耦合度提高业务代码的聚合度。（高内聚、低耦合）
  2. 提高代码复用性
  3. 提高系统扩展性，如高版本兼容低版本
  4. 丰富原本代码功能时，不影响原有代码

AOP的主要应用场景

日志记录、性能统计（功能前后耗时统计、内存统计等）、安全控制、事务处理、实现公共功能的重复使用。

AOP术语

1 横切关注点

从每个方法中抽取出来的同一类非核心业务。如日志和验证参数

2 切面(Aspect)

由关注点所组成的大类，可以有多个切面将自己的关注点切入到目标类上

3 通知(Advice)

每个切面中具体的功能，与关注点类似，可以理解为“关注点在切面中的具体实现就叫通知”，通知相比关注点可以细分。如日志在切面中分成：前置日志，后置日志，报错日志等。。。

4 目标(Target)

也就是目标类，切面中的通知要通知的类

5 代理(Proxy)

向目标对象应用通知之后创建的代理对象

6 连接点(Joinpoint)

横切关注点在程序代码中的具体体现，对应程序执行的某个特定位置。例如：类某个方法调用前、调用后、方法捕获到异常后等。

7 切入点(pointcut)

定位连接点的方式。每个类的方法中都包含多个连接点，所以连接点是类中客观存在的事物。如果把连接点看作数据库中的记录，那么切入点就是查询条件——AOP可以通过切入点定位到特定的连接点。切点通过org.springframework.aop.Pointcut 接口进行描述，它使用类和方法作为连接点的查询条件。

原理分析

AOP思想的底层实现或者说主要思想来源于代理模式，但从细节来说，相较于代理模式又有所优化进步

代理模式

静态代理

静态代理是代理模式中最简单的实现，但是代理类和目标类必须实现同一接口，代理类才能将目标类的目标方法增强执行。而且在编写之初，代理类就已经限定了能够代理的范围，并不灵活。

动态代理

JDK动态代理：

​ jdk自身的动态代理虽然用到了反射技术相比静态代理更为灵活，但是底层实现依旧使用“实现共同接口”这一思想，因此限制了目标类必须实现某一接口，如果一个类没有实现接口，则无法使用jdk自带的动态代理进行代理增强。

CGLIB动态代理：

​ cglib的底层实现原理是通过了继承思想，相比jdk动态代理来说，cglib更加灵活，在目标对象的类没有实现任何接口的前提下，也可以进行代理增强。

AOP的改进

代理模式不论是静态代理还是动态代理，都有一个共同的问题（在静态代理中尤为突出），那就是：若有多种目标对象需要增强（被代理）时，需要创建出多个代理对象，在通过每个代理对象执行目标对象的目标方法。这种情况下，一旦目标对象类型巨大，那么随之产生的代理对象数量也十分巨大。

而AOP面向切面就解决了这一问题，不再使用超多的代理对象，简化了应用开发。
