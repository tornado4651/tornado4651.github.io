2）自动注入

自动注入技术是主要原理是 反射技术 + 注解实现。极大的脱离了xml配置文件，使项目更加简洁易懂

配置环境

配置文件的头部，命名空间添加context规范;并开启自动化注入

使用@Resources自动注入

javax.annotation.Resource是jdk中自带的自动匹配注解，特点：

  1. 默认根据属性字段名与bean标签的id属性值匹配注入
  2. 如果id属性值没有找到，则根据class查找
  3. 可以使用@Resources(name="xxx")自定义匹配bean标签的id属性值
  4. 成员属性可以设置set方法也可以不设置
  5. 该注解可以声明在set方法也可以声明在成员属性上

使用@Autowired自动注入

  1. 默认通过class类型查找bean匹配，与id属性值无关
  2. 若想通过bean标签的id属性值匹配，需要使用org.springframework.beans.factory.annotation.Qualifier注解的value属性另外指名（Qualifier的value属性值 与 bean标签的id属性值匹配）
  3. 成员属性可以有set方法也可以没有
  4. 该注解可以用在成员属性上也可以用在set方法上

注：两种自动注入方法若使用在接口上，借口只有一个实现类bean对象时正常注入，若有多个实现类，则需通过name或value指定对应的bean标签id属性值匹配注入。

Bean的作用域与生命周期

1）bean的作用域

bean的作用域分为 单例作用域 *和 *原型作用域，默认情况下时单例模式。

singleton作用域

1、单例缓存池？

在单例模式中，所有bean对象默认被创建后都放在“单例缓存池中”，每当被调用时从中取出，保证了单例模式的实现。

2、懒加载（lazy-init）？

spring的配置文件中创建bean对象时，可以在通过<bean id="xxx" calss="xxx" lazy-init=true>方法开启懒加载模式。

懒加载表示：Spring在IOC容器启动时不会实例化该bean，而是在程序调用时才去实例化。对于没有设置懒加载的那些bean，Spring在容器初始化时就将那些bean实例化到单例池中以便调用。

默认将lazy-init设置为false的好处：1、程序启动时就能发现配置问题。2、bean对象存于缓存中，使用时无需再次实例化，所以程序更快更有效率。

3、何时使用单例模式（交给spring管理）

当一个bean对象内部的成员变量的属性值不在改变，或者该对象的状态不再改变时，适合对其使用单例模式；当该对象的属性或状态经常改变时（如：user对象）就不适合单例模式。

prototype作用域

spring的ioc容器在启动时会直接实例化原型作用域的bean对象，并不将其放入单例缓存池中，而是每次请求调用都反悔一个新的bean对象。配置文件实现方法：<bean name="xxxx" class="com.xxx.xxx.Xxx" scope="prototype"/>

2）bean的生命周期

bean的生命周期分为四个阶段：bean定义 – bean初始化 – 使用bean – 销毁bean

定义bean

在配置文件中使用<bean>标签定义一个bean对象时，完成bean的定义阶段

bean初始化（init）

方法一：在类中自己创建一个init方法，并在配置文件中的通过init-method属性指名该方法即可

方法二：通过实现InitializingBean这个接口，重写里面的afterPropertiesSet方法，实现自定义初始化bean时的动作。此时无需在配置文件中指定init-method了。

若方法一和方法二同时使用，方法二优先级更高

使用bean对象

通过创建BeanFactory接口的实现类或者ApplicationContext的子类：ClassPathXmlApplicationContext。

销毁bean对象

bean对象的销毁只能通过创建AbstractApplicationContext抽象类后通过close方法关闭容器，进而出发配置文件中指定的destory-method方法。
