2、编写远程调用接口

​ 测试方法：在coupon模块中编写一个接口（接口A：获取对应用户的所有优惠卷），再在用户模块中编写一个页面调用接口（接口B：查看该用户可用的优惠卷），接口B内部调用coupon模块的接口A，获取该用户所有的优惠卷。

第一步：在coupon服务中编写一个远程调用的目标接口，之后member服务将调用该接口，这一步可有可无，因为

第二步：
在member服务中创建一个远程调用接口interface；
使用@FeignClient(【想要调用的目标微服务名称】)对该接口进行注释。
然后在该接口中写明要调用的目标微服务的请求路径与方法名即可（方法名不需要与目标微服务的方法名相同）

第三步：
在主程序启动类上通过@EnableFeignClients(【远程调用接口所在的包路径】)注解开启openfein远程调用

第四步：
在member服务的一个接口中调用coupon接口

配置中心使用方法

1、namespace 命名中心：

最大范围的顶层分层。

用法1：可以分成生产(prod)、测试(test)、开发(dev)多个不同的命名空间，每个命名空间下包含所有服务的配置文件

用法2：为每个服务设置自己专属的命名空间

2、group 配置组：

二级分层，同一命名空间下可有多个不同的分组，配合命名空间实现不同环境的切换。

用法：当命名空间是以服务名进行分配时，可以借助分组实现当前服务命名空间下多种环境的配置隔离（将组名配置为prod . dev . test等）

3、data id 配置集：

最小粒度，即任意一个配置文件本身。

4、确定一个配置文件的坐标

【name space - group - data id】
